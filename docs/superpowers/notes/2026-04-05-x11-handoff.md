## Key Design Decisions (Locked In)

These are in the spec and should not be revisited:

| Decision | Rationale |
|----------|-----------|
| 3-tier architecture (Xvfb → Xpra → VNC) | Xvfb-only for CI, Xpra for interactive, VNC as fallback |
| Python stdlib-only, single file | Copy-into-jail deployment, matches `scripts/claudechic-remote` pattern |
| `start` is declarative (replaces tier set) | Eliminates ambiguity about leftover tiers |
| `stop` is subtractive (removes from set) | `stop --headless` cascades everything |
| `fcntl.flock` for concurrency | Serializes all mutating commands |
| PID validation: alive + comm + etimes (best-effort 3rd factor) | etimes only rejects, never grants; 2-factor is baseline |
| All TCP listeners default to 127.0.0.1 | `--bind-all` for network exposure |
| `xauth add` with `os.urandom` cookie | Display-independent, no live X server needed |
| Stale cleanup: only delete if owner provably gone or not Xvfb | Fail closed on ambiguity |
